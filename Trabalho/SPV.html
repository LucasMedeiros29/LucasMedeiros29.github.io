Objetivo Geral do Sistema

Desenvolver um sistema automatizado baseado em câmeras e técnicas de visão computacional para identificar, classificar e acionar mecanismos de separação de resíduos hospitalares de risco biológico, como seringas, luvas, máscaras, gazes e outros.

Para fins de simplificação e por questões de otimização, serão categorizados apenas quatro objetos, sendo um de cada tipo de classificação de lixo hospitalar.

Técnicas e Ferramentas Utilizadas para o Código

  • OpenCV – Captura de vídeo da webcam, manipulação de imagens, detecção de contornos, desenho de caixas e texto na tela.
  • Python 3 – Linguagem principal utilizada.
  • PyTorch – Criação, modificação e execução da rede neural (ResNet18).
  • Albumentations – Redimensionamento, normalização e conversão de imagens para tensor PyTorch antes da classificação.
  • NumPy – Operações numéricas e manipulação de arrays.
  • Pickle – Carregamento e salvamento de objetos Python.

Desafios Encontrados

Como mencionado, o primeiro desafio encontrado foi o treinamento para identificação dos objetos. Foi tentado utilizar um banco de dados com diversas fotos, resultando em extrema lentidão para conclusão do treinamento e, consequentemente, tornando inviável por hora. Desse modo, o código inicial foi restrito à quatro objetos, um de cada categoria.

Funcionamento

  • Inicialmente, é utilizado o código "auto_calibrate.py" para realizar a calibração da câmera com base , gerando o arquivo final calib.npz;
  • Em seguida, é utilizado o código "record_class" para tirar fotos dos objetos desejados;
  • O código "prepare_dataset" é então utilizado para separar e tratar as imagens, diminuindo seus tamanhos e deixando-as mais leves;
  • O código "train" realiza o treinamento com base nas imagens processadas, gerando arquivos com os dados necessários (pasta models);
  • Por fim, o algoritmo "inference" realiza a classificação em tempo real dos objetos com base nos dados gerados pelo código anterior, exibindo na tela o grau de precisão da medição e a classificação do objeto.

Conclusão

Ao final, os programas estão executando satisfatóriamente o que é proposto, embora melhorias possam ser aplicadas para torná-los mais otimizados e permitir a aplicação em mais objetos. Além disso, também é necessário realizar a implementação da etapa "o que ocorre após a classificação", que está definido, até o momento, como o acionamento do servomotor para manipulação dos objetos.
